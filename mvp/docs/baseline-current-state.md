# FOT Planner MVP - Baseline (рабочая фиксация)

Дата фиксации: 2026-05-25 (обновлено после ревью дроверов)

Цель этого baseline: зафиксировать только стабильное поведение, не включая спорные/недокрученные части drawer.

## 1) Зафиксированная логика индексации

- Индексация применяется с выбранного месяца включительно до декабря.
- Порядок событий в одном месяце:
  1. базоформирующие (`MANUAL_OVERRIDE`, `TARGET_SALARY`, `CLASSIFICATION_CHANGE`, `PLANNED_HIRE`)
  2. `INDEXATION`
  3. завершающие (`TERMINATION`, `TERMINATION_TO_VACANCY`, `CLOSE_POSITION`, `CANCEL_VACANCY`, `TRANSFER`, `POSITION_CARRYOVER`)
- Сортировка применения: `effective_month -> event_priority -> created_order`.
- Удаление события = полный пересчет от seed-значений.
- Массовая индексация выполняется только на `/planning`.
- Факт массовой индексации можно удалить, после удаления выполняется пересчет.

## 2) Что считаем рабочим на текущий момент

### /planning

- KPI и таблицы по позициям (обзор, занятые, вакансии).
- Поиск и фильтрация.
- Массовая индексация:
  - месяц выбирается из списка полных русских названий,
  - есть подтверждение применения,
  - есть явный feedback и журнал применений,
  - есть удаление факта индексации.
- Отображение сотрудника в формате `Имя (Employee ID)`.

### Данные и расчет

- `BONUS` по умолчанию = `0`.
- `CR` как коэффициент (не процент), с цветовой индикацией:
  - `< 0.8` — warning,
  - `0.8..1.2` — ok,
  - `> 1.2` — danger.
- `Dec->Dec` считается по `BASE` (прошлый декабрь BASE -> текущий декабрь BASE).

## 3) Что сознательно НЕ фиксируем как готовое

- Drawer добавления вакансии (в текущем виде нестабилен, дорабатывается отдельно).
- Итоговый UX/компоновка drawer после последних крупных изменений.
- Финальная логика лимитов (новые слоты vs carryover-слоты) — отдельный следующий шаг.
- Финальная логика переходов в части UX и всех крайних кейсов.

## 4) Правило на следующую итерацию

- В baseline считаем «истиной» только поведение из раздела 1 и 2.
- Любые изменения в drawer делаем поэтапно и добавляем в baseline только после отдельного подтверждения.
