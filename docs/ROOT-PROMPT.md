## Подтвержденные продуктовые правила
- Drawer работает по выбранному сотруднику.
- Помесячно в drawer: специализация, уровень, оклад (BASE), премия (BONUS_PLAN), итог ФОТ.
- Индексация применяется ко всем активным строкам с месяца события; если после этого есть ручная правка, дальнейшие индексации считаются уже от нового ручного значения.
- Если сотрудника увольняют с сохранением вакансии, позиция остается в плане как вакансия (IN_LIMIT) и не должна обнуляться по ФОТ.
- Переход сотрудника между позициями/департаментами обязателен как отдельный сценарий (освобождение старой позиции + назначение на новую).
- Дек->Дек %:
  - `prev > 0` -> реальная формула;
  - `0 -> X` -> `100%`;
  - `0 -> 0` -> `0%`.
- Скриншот дизайна — референс, не пиксельный эталон.
