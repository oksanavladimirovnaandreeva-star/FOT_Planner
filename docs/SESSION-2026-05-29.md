## Продуктовые решения (зафиксировать)

### Версии бюджета

| Правило | Деталь |
|---------|--------|
| v1 до утверждения | **Можно править** на `/planning` (статус `DRAFT`, без `publishedAt`) |
| Утверждение v1 | Кнопка «Утвердить бюджет v1» на `/versions` → статус `APPROVED`, дальше v1 только просмотр |
| Квартальный черновик | Полный функционал планирования; создаётся **только после** утверждения v1 |
| Публикация | **Не перезапись** — новая версия **v(N+1)**; предыдущая утверждённая → `ARCHIVED` |
| Согласование (MVP) | Черновик → «Отправить на согласование» (`IN_APPROVAL`) → «Опубликовать v+1» |
| Дек→дек при v+1 | `previousDecemberBase` позиции = декабрь плана **родительской** версии |
| Сравнение | База ↔ черновик: KPI, график по месяцам, отклонения по лимиту/департаменту (референс — скрин BudgetFlow) |

### Прогноз (целевая модель, не полностью в коде)

- **Факт** (из 1С / staging) + **события плана** (пересмотры, переводы, индексация) на оставшийся период.
- Сейчас на `/forecast` только **скелет**: факт YTD + план на будущие месяцы **без** наложения событий.

### Факт (MVP UI)

- Ключ: **`employee_id`** (как `fact.csv` и prod).
- Импорт JSON в «Данные» (`factImport.ts`, образец `mvp/frontend/sample-fact.schema-v1.json`).
- Демо 95% — только в `<details>`, не prod-путь.

### Prod / ИБ (не в MVP UI сейчас)

- Заявка AppSec: master план в приложении, факт из 1С one-way, RBAC/scope на **сервере**, admin без цифр ФОТ, audit, экспорт в контуре видимости.
- **PostgreSQL + API** — после финализации UI и бизнес-логики во frontend.
- Путь A: JSON в браузере → контракт снимка → потом API.

### Drawer (требование пользователя 2026-05-29)

**Нравится идея разделения**, но **текущая двухколоночная вёрстка не лезет** (наложение полей в блоке «События»).

Целевая структура **одним потоком, без скролла всего drawer** (допустим микро-скролл только у таблицы месяцев):

1. Шапка: роль, ID, сотрудник, бейджи, дек→дек, ФОТ.
2. **Один блок «Слот»**: ID · роль · **специализация · уровень** (рядом с позицией).
3. Оргструктура: департамент · юнит · команда.
4. Параметры: с месяца · тип слота · лимит.
5. **Помесячно** (компактная таблица).
6. **События** под таблицей: форма добавления + список с удалением (не отдельная колонка справа).

Не возвращать ID сотрудника в форму — только в шапке.
