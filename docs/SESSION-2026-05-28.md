## Что реализовали в этой сессии

### Safe import (`MvpAppContext`, `AppLayout`)

- `exportCurrentSnapshot`, `inspectSnapshot`, `importCurrentSnapshot` (replace | merge)
- Backup перед импортом: `localStorage` `fot_mvp_pre_import_backup`
- Откат: last export, pre-import backup
- Валидация schema v1, 12 месяцев, дубликаты `positionId` блокируют
- UI: панель «Данные» — preview, confirm, replace/merge, rollback
- Образец: `mvp/frontend/sample-import.schema-v1.json`
- **Не доделано:** warnings в preview, post-merge report, журнал ~5 операций

### KPI / аналитика (`AnalyticsSummaryStrip`)

- **Planning:** одна строка KPI (`singleRow`), без YTD, без avg CR, убран collapsible «Подробная аналитика»
- **Dashboard:** полная лента + fact stub + limit breakdown
- Dec→dec: шапка **% · сумма**; разбивка под ней **сумма · %**
- Карточки Позиции / Вакансии: `кол-во · сумма` (без слова «вакансий» в строках разбивки)
- Вёрстка: сайдбар ~176px, max-width контента 92rem, KPI как мини-карточки с рамкой

### Planning

- Фильтры в **одну компактную строку**: поиск, департамент, юнит, команда, лимит, статус
- Массовая индексация по **отфильтрованным** незакрытым позициям
- Carryover-панель для вакансий «перенос с прошлого года»

### Drawer / вакансии

- Новая вакансия наследует батчи индексации (`applyExistingIndexationBatches`)
- Черновик: события через `onSaveDraft(..., forceCreate=true)`
- Удаление вакансии в drawer с confirm
- `PositionDrawer.baseline.tsx` синхронизирован с `PositionDrawer.tsx`
